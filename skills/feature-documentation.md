# Template: Documentação de Feature

## Estrutura do Documento

### 1. Resumo Executivo
- Descrição concisa da feature
- Problema que resolve
- Valor entregue
- Status atual

### 2. Casos de Uso
Para cada caso de uso principal:
- Nome descritivo
- Ator
- Pré-condições
- Fluxo
- Pós-condições
- Exemplo prático

### 3. Implementação Técnica
- Diagrama de arquitetura/fluxo (Mermaid)
- Tabela de componentes criados/modificados
- Stack técnico
- Padrões de design

### 4. Guia de Uso

#### Para Desenvolvedores
- Instalação e setup
- Código de exemplo
- Configurações necessárias

#### Para Usuários Finais (se aplicável)
- Passo a passo de uso
- Screenshots ou exemplos

### 5. API/Interface Pública
- Tabela de endpoints/métodos
- Exemplos de requisição/resposta
- Parâmetros e retornos

### 7. Considerações
- Links para issues/tickets
- Documentação de bibliotecas
- RFCs ou ADRs relacionados
- Compatibilidade
